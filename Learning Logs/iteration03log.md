The biggest feedback that I got was that the Name/Label input was confusing for the Cash asset/dropdown. To fix this, I added an if/else code to hide the input. Next they suggested I add a delete all button on the saved snapshots, rather than only having a delete for each out. The button used the function localStorage.removeItem and deleteAllBtn.addEventListener. It also used an if statement to give the user a warning that told them the delete could not be undone. Lastly, they told me to make the look more professional. Rather than a reddit site or video game site, they siggested it looked more like a professional financial tool. I made the background lighter, changed the fonts, and made some dark green accents to achieve this. The site is simple enough that there are not many edge cases -- it relies heavily on user labeling, but the output is also the same as these labels. The user can enter whatever they want and have consistent results. With more time, I would add integration into banking apps and live tracking of asset values so the user did not have to input them constantly.
